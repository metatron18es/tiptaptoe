Comentarios

  • HTML

    El código HTML ha sido reelaborado con el objetivo de mejorar su semántica y organizarlo de manera más eficiente.

    Adicionalmente, se ha incorporado un panel lateral que permite la visualización de comentarios relacionados con el desarrollo, así como los motivos que respaldan las modificaciones realizadas.

  • CSS

    La hoja de estilos CSS ha sido completamente reescrita, adoptando el uso de SCSS.

    Ahora mismo, la hoja de estilos se beneficia tanto de las mejoras presentes en el estándar CSS actual (propiedades personalizadas, funciones como calc(), entre otras) como de las ventajas proporcionadas por SCSS (mixins, variables, funciones, etc.).

    Se contempla la posibilidad de mejorar esta sección en el futuro mediante su división en archivos independientes.

  • JS

    El código JavaScript ha sido reescrito, preservando parte del desarrollo original mientras se optimiza la lógica del juego.

    Se ha optado por separar la lógica de presentación (DOMManipulator) de la lógica del juego (TicTacToeGame), siguiendo el principio de responsabilidad única.

    El uso de clases y propiedades de clase ha permitido encapsular la lógica del juego, dividiendo funciones más extensas en fragmentos más específicos para mejorar la legibilidad y mantenibilidad.

    Se han dividido las funciones más grandes en funciones más pequeñas y específicas para mejorar la legibilidad y la mantenibilidad.

    Se han empleado funciones más descriptivas y claras para mejorar la comprensión del código, así como funciones flecha para simplificar la estructura y evitar posibles problemas con el contexto de "this".

    A pesar de estas mejoras, aún se pueden abordar otras áreas de optimización, tales como:

    • Utilización de Clases y Módulos: División de la lógica del juego y la manipulación del DOM en clases separadas para una gestión y organización más efectiva del código. Además, considerar el uso de módulos si el código sigue creciendo.
    • Empleo de CSS en lugar de Clases en JavaScript: Donde sea posible, evitar la adición y eliminación directa de clases en JavaScript, optando en su lugar por controlar el aspecto visual a través de CSS y modificar solo las propiedades necesarias desde JavaScript.
    • Utilización de Funciones Puras: Se recomienda mantener las funciones lo más puras posible, evitando la manipulación directa del DOM y optando por devolver valores necesarios para su posterior manipulación.
    • Desacoplamiento del Juego de la Interfaz de Usuario: Separar la lógica del juego de la interfaz de usuario para facilitar la reutilización del código y permitir la implementación de diferentes interfaces de usuario si es necesario.
    • Mejora de la Estructura de Datos: Considerar el uso de una clase para representar el estado del juego en lugar de matrices bidimensionales, con el objetivo de hacer las operaciones relacionadas con el juego más intuitivas y comprensibles.
